# Entra PIM Manager 0.2.0

Sign-in for federated tenants, a simpler add-account flow, and quicker access to logs.

## New

- **Device-code sign-in fallback.** For tenants whose UPN domain is federated to an external identity provider (e.g. Okta) with aggressive seamless SSO — where the normal sign-in would silently bind the app to the wrong (Office) account — you can now sign in with a device code and complete it on a second device (e.g. your phone). Available under **Add account → Advanced**.
- **One "Add account" screen.** The separate "Add account", "Connect to another tenant", and device-code entry points are merged into a single slide-in. The tenant field is optional (leave it blank for your home tenant); the device-code option lives under **Advanced**.
- **Open log folder from Settings.** A new **Diagnostics** section adds an "Open log folder" button so you can grab the log files when reporting a problem.

## Improved

- **Clearer sign-in errors.** When an app registration isn't configured for desktop sign-in, the app now explains the fix ("enable *Allow public client flows*") instead of showing a generic error.
- **Setup documentation** clarifies when "Allow public client flows" is required: the broker sign-in works without it, but the device-code flow needs it.

## Notes

- The device-code flow runs without the Windows broker, so a Conditional Access policy that requires a managed device — or that blocks device-code flow — will reject it.
- Existing enrolled accounts keep working unchanged; they continue to use broker (WAM) sign-in.
